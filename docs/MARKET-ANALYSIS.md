## 二、功能趋势分析

### 已成标配（必须有）
- 页面对话 / 总结 / 划词解释
- 多模型支持（GPT-4/Claude/DeepSeek）
- 侧边栏 UI
- 右键菜单集成

### 差异化竞争点
- **知识管理深度** — Readwise/Obsidian 的护城河
- **视频/音频总结** — Sider/Monica 的流量入口
- **OCR/图片理解** — 多模态能力
- **写作辅助** — Monica 的变现手段
- **本地优先/隐私** — Obsidian 的用户忠诚度来源

### 市场空白（没人做好）
- **知识积累 + AI 理解的闭环** — 保存了但从不复习
- **个性化学习** — AI 不知道你的水平和偏好
- **跨会话知识关联** — 每次对话都是独立的
- **开发者向深度功能** — 代码审查、API 文档理解

## 三、用户真正想要的（但现有工具做不好）

### 1. "别让我离开当前页面"
用户浏览时遇到不懂的内容，最怕的是中断 flow。所有操作应该在侧边栏/浮动面板内完成。

### 2. "记住我学过什么"
现有工具只管回答，不管复习。用户今天问了一个问题，下周遇到类似问题又忘了。

### 3. "别给我泛泛而谈"
AI 回答太通用，不知道用户的技术水平。初学者和专家需要不同深度的回答。

### 4. "帮我把碎片知识串起来"
看了 10 篇文章，每篇都问了问题，但知识是散的。用户需要知识图谱/关联。

### 5. "我关心隐私"
API Key、浏览内容、知识库都应该在本地。不信任云端。

### 6. "别收我订阅费"
Chrome 扩展订阅疲劳。一次买断或免费 + 自带 API Key 的模式更受欢迎。

## 四、对智阅 PageWise 的启示

### 应该补的（标配功能）
1. ✅ 划词提问（已完成）
2. 页面高亮标注
3. YouTube/视频字幕总结
4. 对话历史按 URL 持久化
5. 快捷键支持

### 应该做深的（差异化）
1. 知识关联 — 自动关联历史知识条目
2. 间隔复习 — 基于 Anki 算法的知识卡片
3. 用户画像驱动的个性化回答
4. 技能系统持续扩展
5. 本地优先 + 离线可用

### 应该做新的（市场空白）
1. **知识图谱可视化** — 展示知识之间的关联
2. **学习路径生成** — 基于知识库生成个性化学习路线
3. **多页面联合分析** — 同时分析多个标签页的内容
4. **API 文档专用模式** — 自动提取端点、生成示例代码
5. **代码仓库理解** — GitHub/GitLab 页面深度分析
