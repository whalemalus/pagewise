### 评分标准

#### 需求符合度（30%）
| 分数段 | 标准 |
|--------|------|
| 90-100 | 所有需求点完整实现，边界情况处理完善 |
| 80-89 | 核心功能实现，少数边界情况未覆盖 |
| 70-79 | 主要功能实现，但有明显遗漏 |
| <70 | 功能不完整，核心需求未满足 |

#### 代码质量（25%）
| 分数段 | 标准 |
|--------|------|
| 90-100 | 代码清晰、JSDoc 完善、无重复、设计模式合理 |
| 80-89 | 代码可读、关键注释有、偶有重复 |
| 70-79 | 代码可读但有坏味道、注释不足 |
| <70 | 代码混乱、难以维护 |

#### 安全性（20%）
| 分数段 | 标准 |
|--------|------|
| 90-100 | 无硬编码密钥、输入验证完善、无注入风险 |
| 80-89 | 基本安全，有轻微风险需注意 |
| 70-79 | 存在安全隐患，需修复 |
| <70 | 有严重安全漏洞 |

#### 性能（15%）
| 分数段 | 标准 |
|--------|------|
| 90-100 | 算法高效、懒加载、无内存泄漏 |
| 80-89 | 性能良好，有小优化空间 |
| 70-79 | 有明显性能问题 |
| <70 | 性能严重不足 |

#### 测试覆盖（10%）
| 分数段 | 标准 |
|--------|------|
| 90-100 | 测试数增加或保持、关键路径有测试、全部通过 |
| 80-89 | 测试数不变、基本覆盖 |
| 70-79 | 测试数减少但核心功能仍有覆盖 |
| <70 | 测试数大幅减少或核心功能无测试 |
