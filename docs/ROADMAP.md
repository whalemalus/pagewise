# 30 轮飞轮迭代路线图 — 智阅 PageWise

> Plan Agent 制定，2026-04-26
> 基于竞品分析 + 用户需求洞察

---

## 路线图总览

```
Phase 1: 补齐标配 (迭代 4-8)    → 让基础体验完整
Phase 2: 差异化建设 (迭代 9-14) → 建立竞争壁垒
Phase 3: 知识系统深化 (15-20)   → 让知识活起来
Phase 4: 高级能力 (21-26)       → 做别人做不到的
Phase 5: 打磨发布 (27-33)       → 准备上架 Chrome 商店
```

## Phase 1: 补齐标配（迭代 4-8）

> 目标：让用户觉得"该有的都有了"

### 迭代 4 — 页面高亮标注
- **需求**: 用户选中文本后可以高亮保存，下次访问同一页面时高亮仍在
- **实现**: content script + chrome.storage 存储高亮位置（XPath + offset）
- **价值**: 对标 Readwise/Liner 的核心功能

### 迭代 5 — 对话历史持久化增强
- **需求**: 对话按 URL 保存，重新访问同一页面时恢复对话
- **实现**: 将 conversationHistory 与 URL 关联存储到 IndexedDB
- **价值**: 用户体验连贯性

### 迭代 6 — 快捷键系统
- **需求**: 键盘快捷键快速操作（打开侧边栏、总结页面、清空对话等）
- **实现**: manifest.json 注册 commands + background 监听
- **快捷键**: Ctrl+Shift+Y 打开侧边栏, Ctrl+Shift+S 总结页面
- **价值**: 效率提升，对标 Sider/Monica

### 迭代 7 — 导出增强：对话导出
- **需求**: 将当前对话导出为 Markdown/JSON
- **实现**: sidebar.js 添加导出对话功能
- **价值**: 用户可以将有价值的对话保存到笔记工具

### 迭代 8 — 页面内容预览面板
- **需求**: 用户可以看到 AI 实际"看到"了什么内容
- **实现**: 在侧边栏添加一个"页面内容"标签，显示提取的内容摘要
- **价值**: 透明度，帮助用户理解 AI 为什么这样回答

## Phase 2: 差异化建设（迭代 9-14）

> 目标：建立"别人没有但我有"的能力

### 迭代 9 — YouTube 视频字幕总结
- **需求**: 打开 YouTube 视频页面时，自动提取字幕并支持总结
- **实现**: page-sense 识别 YouTube 页面，content script 提取字幕数据
- **价值**: 流量入口，Sider/Monica 的核心功能

### 迭代 10 — 多页面联合分析
- **需求**: 用户可以选择多个标签页，让 AI 同时分析
- **实现**: background script 收集多标签页内容，合并后发给 AI
- **价值**: 市场空白，竞品都没有

### 迭代 11 — 技能系统扩展：自定义技能
- **需求**: 用户可以创建自己的技能（类似 GPTs）
- **实现**: 技能编辑器 UI + 用户技能存储
- **价值**: 差异化，让 PageWise 变成平台

### 迭代 12 — API 文档专用模式
- **需求**: 识别 API 文档页面，自动提取端点列表、参数、示例
- **实现**: page-sense 增加 API 文档识别 + 专用提取逻辑
- **价值**: 开发者向核心功能

### 迭代 13 — GitHub 仓库页面理解
- **需求**: 打开 GitHub 仓库时，自动分析 README、目录结构、关键文件
- **实现**: page-sense 识别 GitHub 页面 + 专用提取
- **价值**: 开发者向核心功能

### 迭代 14 — 代码执行沙箱
- **需求**: AI 回答中的代码可以直接在侧边栏运行（HTML/JS）
- **实现**: iframe sandbox 执行，显示输出结果
- **价值**: 学习闭环，写了就能试

## Phase 3: 知识系统深化（迭代 15-20）

> 目标：让保存的知识"活"起来

### 迭代 15 — 知识关联引擎
- **需求**: 新保存的知识自动关联历史相似条目
- **实现**: 基于 TF-IDF 或简单向量相似度，自动打关联标签
- **价值**: 知识不再是孤岛

### 迭代 16 — 间隔复习系统
- **需求**: 基于 Anki SM-2 算法，定期推送知识卡片复习
- **实现**: 知识条目添加复习调度数据，sidebar 显示"今日复习"
- **价值**: 对标 Readwise，学习效果倍增

### 迭代 17 — 知识图谱可视化
- **需求**: 可视化展示知识条目之间的关联关系
- **实现**: Canvas/SVG 力导向图，节点=知识条目，边=关联
- **价值**: Obsidian 用户最爱的功能

### 迭代 18 — 智能搜索增强
- **需求**: 知识库支持语义搜索（不只是关键词匹配）
- **实现**: 本地 embedding + 余弦相似度（用 ONNX.js 或纯 JS）
- **价值**: 知识检索体验质的飞跃

### 迭代 19 — 学习路径生成
- **需求**: 基于知识库内容，AI 自动生成个性化学习路线图
- **实现**: 分析知识库中的标签和主题，调用 AI 生成路径
- **价值**: 市场空白，从"工具"升级为"导师"

### 迭代 20 — 知识库批量管理
- **需求**: 批量选择、批量删除、批量打标签、批量导出
- **实现**: 知识列表添加复选框 + 批量操作栏
- **价值**: 知识量大了以后的刚需

## Phase 4: 高级能力（迭代 21-26）

> 目标：做别人做不到的

### 迭代 21 — 多模态：图片理解
- **需求**: 页面中的图片可以被 AI 理解（截图、图表、流程图）
- **实现**: 提取页面图片 URL，发送给支持 vision 的模型
- **价值**: 多模态是趋势

### 迭代 22 — 多模态：PDF 阅读
- **需求**: 在浏览器中打开 PDF 时，PageWise 可以阅读和问答
- **实现**: PDF.js 提取文本 + content script 适配
- **价值**: 学术/技术文档场景

### 迭代 23 — Prompt 模板库
- **需求**: 用户可以保存常用的 Prompt 模板，一键调用
- **实现**: 模板管理 UI + 快捷插入
- **价值**: 效率提升

### 迭代 24 — 对话分支
- **需求**: 从对话的任意节点分叉，尝试不同的提问方向
- **实现**: 对话树结构，支持从任意消息创建分支
- **价值**: 深度探索场景

### 迭代 25 — 上下文窗口管理
- **需求**: 显示当前 Token 用量，支持手动选择包含哪些上下文
- **实现**: Token 计数器 + 上下文选择器
- **价值**: 高级用户的需求

### 迭代 26 — 数据统计仪表盘
- **需求**: 使用统计（提问次数、知识条目数、常用技能、Token 消耗）
- **实现**: 统计面板 UI + 数据聚合
- **价值**: 用户粘性

## Phase 5: 打磨发布（迭代 27-33）

> 目标：准备上架 Chrome Web Store

### 迭代 27 — 性能优化
- **需求**: 大知识库（1000+条）下的性能
- **实现**: 虚拟滚动、懒加载、索引优化
- **价值**: 用户体验

### 迭代 28 — 错误处理全面升级
- **需求**: 所有边界情况的友好错误提示
- **实现**: 全局错误捕获 + 用户友好提示 + 重试机制
- **价值**: 稳定性

### 迭代 29 — 引导流程 + 新手教程
- **需求**: 首次安装的引导流程
- **实现**: 分步引导（配置 API → 试用功能 → 保存知识）
- **价值**: 降低使用门槛

### 迭代 30 — Chrome 商店准备
- **需求**: 隐私政策、商店描述、截图、权限说明
- **实现**: 文档 + 截图 + manifest 权限最小化
- **价值**: 正式发布

### 迭代 31 — 国际化 (i18n)
- **需求**: 支持英文界面
- **实现**: i18n 框架 + 语言文件
- **价值**: 全球用户

### 迭代 32 — 可访问性 (a11y)
- **需求**: 屏幕阅读器支持、键盘导航
- **实现**: ARIA 标签、focus 管理
- **价值**: 包容性

### 迭代 33 — 最终测试 + 发布
- **需求**: 端到端测试、性能测试、安全审计
- **实现**: 全面测试 + 修复 + 打包发布
- **价值**: 正式上线

## Phase 6: LLM Wiki 知识编译系统（迭代 34-40）

> 目标：基于 Karpathy LLM Wiki 模式，将 PageWise 从「存储答案」升级为「编译知识」
> 参考: https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f

### 迭代 34 — Level 1.1: 知识库 Markdown 导出
- **需求**: 将 IndexedDB 中的 Q&A 条目导出为 LLM Wiki 格式的 Markdown 文件
- **实现**: 每条 Q&A → 独立 .md 文件（含 YAML frontmatter），自动生成 index.md
- **价值**: 打通 PageWise → Claude Code 的知识流转

### 迭代 35 — Level 1.2: 实体/概念自动提取
- **需求**: 导出时用 AI 自动识别 Q&A 中的实体和概念，生成独立 wiki 页面
- **实现**: AI 提取实体/概念 → 生成 entities/ 和 concepts/ 页面
- **价值**: 知识从「Q&A 列表」变为「结构化百科」

### 迭代 36 — Level 1.3+1.4: 交叉引用 + Git 集成
- **需求**: 导出的页面自动建立 [[wikilinks]]，导出目录自动 Git 管理
- **实现**: 实体匹配 + 标签关联 → 双向链接；每次导出自动 commit
- **价值**: 知识形成网络，版本可追溯

### 迭代 37 — Level 2.1+2.2: 知识编译引擎
- **需求**: Q&A 自动分类（实体/概念维度），基于实体共现建立深度关联
- **实现**: IndexedDB schema 扩展 + 实体共现分析 + 关联强度评分
- **价值**: 知识不再是孤岛

### 迭代 38 — Level 2.3+2.4+2.5: 矛盾检测 + 编译报告
- **需求**: 新回答与已有知识冲突时提示，每次 ingest 生成编译报告
- **实现**: 同主题 Q&A 对比 + 事实性矛盾检测 + 增量编译
- **价值**: 知识质量保障

### 迭代 39 — Level 3.1+3.2: Wiki 浏览 + 图谱增强
- **需求**: 侧边栏新增 Wiki 浏览标签，知识图谱增加 wiki 视图
- **实现**: Wiki 标签页 + [[wikilinks]] 跳转 + 节点类型区分
- **价值**: 在浏览器中直接浏览 wiki

### 迭代 40 — Level 3.3+3.4+3.5+3.6: Ingest + Query + Lint + 同步
- **需求**: 一键编译进 wiki、Ask Wiki 模式、Wiki 健康检查、服务器同步
- **实现**: 自动 ingest 流程 + wiki 级别问答 + lint 工具 + WebDAV/Git 同步
- **价值**: 完整的 LLM Wiki 浏览器体验

---

## 执行原则

1. **每轮迭代只做 1-2 个功能**，避免 Sub Agent 超限
2. **Guard 审核必须通过**才能进入下一轮
3. **测试必须全绿**才能提交
4. **文档必须同步**更新
5. **失败归因**：每轮迭代结束后，如果有问题，记录到 VERIFICATION.md 并在下一轮修复
