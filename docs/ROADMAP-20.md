# 20 轮飞轮迭代计划 — 智阅 PageWise

> 创建日期: 2026-04-29
> 目标: 性能优化 + 易用性提升 + 系统进化
> 当前状态: v1.5.1, 14,287 LOC, 134 tests, 0 failures

---

## Phase 1: 性能优化 (R1-R5)

### R1: sidebar.js 代码拆分 — 消息系统
- **目标**: 将 sidebar.js (6,553行) 拆分，消息相关代码提取到 `lib/message-renderer.js`
- **范围**: `addAIMessage()`, `addUserMessage()`, `addSystemMessage()`, `updateAIMessage()`, `showLoading()`, `handleMessageAction()`, `renderMarkdown()` 调用
- **验收**: sidebar.js 减少 500+ 行，测试通过，消息渲染不变

### R2: sidebar.js 代码拆分 — 知识面板
- **目标**: 知识库 UI 代码提取到 `lib/knowledge-panel.js`
- **范围**: `loadKnowledgeList()`, `showKnowledgeList()`, `renderKnowledgeList()`, `searchKnowledge()`, `batchDelete()`, `batchTag()`, `batchExport()` 等
- **验收**: sidebar.js 再减少 800+ 行，测试通过

### R3: 知识库搜索索引优化
- **目标**: 为 IndexedDB 添加全文搜索索引，避免每次搜索遍历全部条目
- **范围**: knowledge-base.js 的 `search()` 方法，引入倒排索引
- **验收**: 1000+ 条目搜索 < 100ms（当前可能 > 500ms）

### R4: 虚拟滚动 — 知识列表
- **目标**: 知识列表只渲染可视区域的条目
- **范围**: `appendKnowledgePage()` + `setupInfiniteScroll()` 改为虚拟滚动
- **验收**: 10,000 条目列表滚动流畅 (60fps)

### R5: 消息气泡懒渲染
- **目标**: 聊天区域的历史消息只渲染最近 50 条，滚动时动态加载
- **范围**: `chatArea` 的消息渲染逻辑
- **验收**: 100+ 条消息的对话不卡顿

---

## Phase 2: 易用性提升 (R6-R10)

### R6: 停止生成按钮
- **目标**: AI 回答生成中可以随时停止
- **范围**: `sendMessage()` 中 AbortController，`btnStop` 按钮逻辑
- **验收**: 点击停止按钮立即中断流式输出

### R7: 键盘快捷键
- **目标**: 常用操作支持快捷键
- **范围**: Ctrl+Enter 发送, Esc 清空输入, Ctrl+K 搜索, Ctrl+N 新对话
- **验收**: 所有快捷键可用且有提示

### R8: 智能错误处理
- **目标**: API 错误分类 + 一键重试 + 自动降级
- **范围**: `handleAIError()` 增强，区分网络错误/限流/认证失败/模型过载
- **验收**: 每种错误有明确提示和操作建议

### R9: 代码块增强
- **目标**: 语法高亮 + 一键复制 + 代码预览
- **范围**: `renderMarkdown()` 的代码块渲染，引入 Prism.js 或 highlight.js (CDN)
- **验收**: 常见语言有语法高亮，复制按钮可用

### R10: 对话历史管理
- **目标**: 查看/恢复/删除历史对话
- **范围**: 新增对话列表面板，IndexedDB 对话存储
- **验收**: 可以浏览历史对话并恢复

---

## Phase 3: 系统进化 (R11-R15)

### R11: 知识图谱可视化
- **目标**: 以图谱形式展示知识条目之间的关联
- **范围**: 基于标签/引用关系构建图谱，Canvas 或 SVG 渲染
- **验收**: 可以看到知识条目之间的关联网络

### R12: 间隔重复复习系统
- **目标**: 基于遗忘曲线自动推送待复习的知识
- **范围**: `lib/spaced-repetition.js` 增强，复习面板 UI
- **验收**: 每天推送 3-5 条待复习知识

### R13: 页面截图 + OCR
- **目标**: 支持截图提问，AI 分析图片内容
- **范围**: `chrome.tabs.captureVisibleTab()` + 图片发送给 AI
- **验收**: 截图后可以向 AI 提问关于图片的内容

### R14: 知识统计仪表盘
- **目标**: 可视化学习统计：知识增长曲线、主题分布、学习热力图
- **范围**: `lib/stats.js` 增强 + 统计面板 UI
- **验收**: 展示知识条目数/标签数/学习天数/增长趋势

### R15: 多语言知识库
- **目标**: 支持英文页面提取 + 中英双语知识库
- **范围**: 页面提取语言检测，知识条目标记语言，搜索跨语言
- **验收**: 英文页面正确提取，知识库支持中英双语

## Phase 4: 打磨发布 (R16-R20)

### R16: 首次使用引导
- **目标**: 新用户首次打开有引导流程
- **范围**: `lib/onboarding.js` 增强，API 配置向导 + 功能介绍
- **验收**: 新用户 3 分钟内完成配置并提问

- **目标**: 内置性能指标收集，帮助诊断卡顿
- **范围**: `lib/log-store.js` 增加性能指标，日志面板增强
- **验收**: 可以看到 API 响应时间、渲染耗时、内存使用

### R18: 设置导入导出
- **目标**: 支持导出/导入全部设置 + 知识库
- **范围**: 设置序列化，一键备份/恢复
- **验收**: 导出 zip 文件包含设置 + 知识库，可导入恢复

### R19: Chrome Web Store 准备
- **目标**: 完善 manifest.json 权限声明、隐私政策、图标资源
- **范围**: 权限最小化审查、隐私政策文档、图标多尺寸
- **验收**: 满足 Chrome Web Store 审核要求

### R20: v2.0 发布
- **目标**: 最终测试 + 性能基准 + 版本发布
- **范围**: 全量测试、性能基准记录、CHANGELOG 整理、GitHub Release
- **验收**: 0 测试失败，性能达标，v2.0 release 发布
