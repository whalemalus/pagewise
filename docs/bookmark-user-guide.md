## 书签管理基础

### 添加书签

- **手动添加** — 在浏览器中打开目标页面，点击 PageWise 图标或使用快捷键
- **批量导入** — 支持从 Chrome 书签 HTML 文件和 JSON 格式导入

### 书签状态

每条书签有三种状态：
- `unread` — 未读
- `reading` — 阅读中
- `read` — 已读

状态可在详情面板中手动切换，帮助你追踪阅读进度。

### 标签管理

- **手动标签** — 在标签编辑器中添加/删除/重命名标签
- **自动标签** — 基于 URL 模式和页面标题自动推荐标签
- 标签支持中文和英文关键词

### 内容预览

每条书签可生成内容摘要预览（最长 200 字），支持可配置选项：
- 包含标签信息
- 包含状态信息
- 包含文件夹路径

### 文件夹分析

- 自动分析书签文件夹结构
- 评估文件夹质量（空文件夹检测、嵌套深度）
- 生成文件夹优化建议

## 智能搜索

### 综合搜索

PageWise 提供多层次搜索能力：

| 搜索类型 | 说明 |
|---------|------|
| **关键词搜索** | 基于倒排索引的快速全文匹配 |
| **语义搜索** | 基于 TF-IDF 向量的语义相关性搜索 |
| **混合搜索** | 关键词 + 语义加权合并（默认权重 0.6:0.4）|
| **以文搜文** | 选择一条书签，查找语义相似的其他书签 |

### 搜索过滤

可组合使用以下过滤条件：
- **文件夹** — 按文件夹路径过滤
- **标签** — 按标签数组过滤
- **状态** — 按阅读状态过滤
- **域名** — 按网站域名过滤
- **排序** — 按相关性 / 日期 / 标题排序
- **数量限制** — 限制返回结果数（默认 20）

### 搜索建议

- 基于标签和热门搜索自动生成建议
- 支持 200ms 防抖，避免频繁请求

### 语义搜索权重

搜索时各字段的权重：
| 字段 | 权重 |
|------|------|
| 标题 (title) | 3.0 |
| 标签 (tags) | 2.0 |
| 内容预览 (contentPreview) | 1.5 |
| 文件夹路径 (folderPath) | 1.0 |
| URL | 0.5 |

## 重复书签检测与清理

### 三种去重策略

1. **URL 完全去重** — 规范化 URL 后精确匹配
   - 自动移除协议前缀 (http/https)
   - 自动去除跟踪参数 (utm_source, fbclid, gclid 等)
   - 移除尾部斜杠
2. **标题相似度去重** — Jaccard 系数 > 0.7（可配置阈值）
3. **综合去重** — 合并以上两种策略

### 清理建议

- `suggestCleanup()` 生成合并或删除建议
- `batchRemove()` 批量清理重复书签

## AI 智能推荐

### 推荐类型

| 类型 | 说明 |
|------|------|
| **pattern** | 基于收藏模式的学习建议 |
| **gap-filling** | 知识盲区领域的入门资源 |
| **depth** | 已学领域的进阶方向 |

### 难度分类

- **入门 (beginner)** — 教程、快速上手、基础
- **中级 (intermediate)** — 最佳实践、实战、技巧
- **高级 (advanced)** — 架构、性能优化、源码分析

### 工作流程

1. 分析用户收藏模式（收藏画像）
2. 检测知识盲区（Gap Detector）
3. 调用 LLM 生成个性化推荐
4. 推荐结果缓存 30 分钟

### 降级策略

当 AI 服务不可用时，自动降级到规则引擎推荐（BookmarkRecommender）。

## 书签图谱

### 图谱引擎

BookmarkGraphEngine 从书签数组构建相似度图谱：
- **混合相似度算法** — Jaccard + 域名匹配 + 文件夹重叠
- **Top-K 推荐** — 找到最相似的 K 条书签
- **聚类视图** — 按域名/文件夹自动聚类
- **性能优化** — 倒排索引避免 O(n²) 全量计算，1000 书签 < 10 秒

### 可视化

- Canvas 画布渲染节点和边
- 支持缩放、平移
- 点击节点查看书签详情

### 详情面板

- 显示书签完整信息
- 相似书签列表
- 快速操作（编辑标签、修改状态）

## 智能集合

BookmarkSmartCollections 提供基于规则的动态集合：

### 内置集合

- **未读** — 所有 `unread` 状态的书签
- **最近添加** — 最近添加的书签
- **本周阅读** — 本周内阅读过的书签

### 自定义集合

支持以下规则类型（AND 逻辑组合）：
- 标签 (tags)
- 域名 (domain)
- 文件夹 (folder)
- 状态 (status)
- 时间范围 (dateRange)
- 分类 (category)

集合规则支持序列化/反序列化，可持久化保存。

## 导入导出与分享

### 导入

| 格式 | 方法 | 说明 |
|------|------|------|
| Chrome HTML | `importFromChromeHTML(html)` | 解析 Chrome 书签 HTML |
| JSON | `importFromJSON(json)` | 完整图谱数据导入 |

### 导出

| 格式 | 方法 | 说明 |
|------|------|------|
| JSON | `exportJSON()` | 完整图谱数据（书签+聚类+标签+状态） |
| CSV | `exportCSV()` | 书签列表 CSV（含表头） |
| 文件 | `exportToFile(format)` | 导出为 Blob（`'json'` 或 `'csv'`） |

### 分享

- **创建分享集合** — 从智能集合创建可分享数据
- **隐私控制** — 可选移除个人信息（文件夹路径、标签、状态）
- **URL 匿名化** — 将 URL 替换为域名摘要
- **导出格式** — JSON / 纯文本摘要 / Base64 编码
- **分享链接** — 生成 `data:` URI 分享链接

## 错误处理与安全

### 错误分类

BookmarkErrorHandler 将错误分为 5 类：
- **network** — 网络连接、超时、DNS 问题
- **permission** — 权限不足、未授权
- **storage** — 存储配额、读写异常
- **validation** — 数据校验失败
- **unknown** — 未分类错误

### 优雅降级

- 错误发生时提供恢复建议
- `createErrorBoundary()` 为异步函数创建安全包装
- 结构化日志记录所有错误上下文

### 安全审计

BookmarkSecurityAudit 对扩展进行安全检查：
- **权限审计** — 检测过度权限请求和危险权限
- **内容脚本审计** — 检测 `<all_urls>` 和不安全配置
- **CSP 审计** — 检测缺失 CSP、`unsafe-eval`、`unsafe-inline`

### 性能优化

BookmarkPerformanceOptimizer 提供：
- 分批处理（默认每批 500 条）
- LRU 缓存淘汰
- 视口裁剪（虚拟化渲染）
- Worker 卸载支持

## 故障排除

### 书签未显示

1. 确认扩展已启用（`chrome://extensions/`）
2. 检查权限是否已授予（`bookmarks` 权限）
3. 刷新扩展页面或重启浏览器

### 搜索无结果

1. 确认索引已构建（首次使用需等待索引完成）
2. 尝试使用更短或更通用的关键词
3. 检查过滤条件是否过于严格

### AI 推荐不工作

1. 确认网络连接正常
2. AI 推荐依赖 LLM 服务，检查 API 密钥配置
3. 服务不可用时会自动降级到规则引擎推荐

### 导入失败

1. Chrome HTML 导入：确认文件是 Chrome 导出的标准书签 HTML 格式
2. JSON 导入：确认 JSON 格式正确，包含 `bookmarks` 数组
3. 大文件导入会分批处理，显示进度回调

### 快捷键冲突

1. 在设置面板中查看当前快捷键绑定
2. 使用冲突检测功能识别重叠绑定
3. 自定义快捷键或重置为默认值

### 性能问题

1. 万级书签场景下，图谱构建会自动分批处理
2. 清理重复书签可减少数据量
3. 启用 Worker 卸载可避免主线程阻塞

## FAQ

### Q: PageWise 支持哪些浏览器？

A: 目前主要支持 Chrome (Manifest V3)。正在探索 Firefox 和 Edge 的兼容性支持。

### Q: 书签数据存储在哪里？

A: 书签数据存储在浏览器本地：
- 书签元数据和图谱数据使用 `chrome.storage`
- 学习进度数据使用 IndexedDB
- 所有数据保留在本地，不会上传到服务器（除 AI 推荐外）

### Q: AI 推荐会泄露我的书签数据吗？

A: AI 推荐调用 LLM 服务时会发送收藏画像摘要（非原始书签列表）。你可以：
- 配置隐私过滤（移除个人信息）
- 选择不使用 AI 推荐，使用规则引擎推荐替代

### Q: 如何备份我的数据？

A: 使用导出功能将数据导出为 JSON 文件。建议定期导出备份。

### Q: 如何分享书签给其他人？

A: 使用分享功能创建分享集合，支持：
- 隐私过滤（移除个人数据）
- 多种导出格式（JSON / 纯文本 / Base64）
- 生成分享链接

### Q: 书签去重会影响原始数据吗？

A: `suggestCleanup()` 仅生成建议，不会自动删除。`batchRemove()` 需要手动确认执行。建议先导出备份再执行批量清理。

### Q: 支持多少条书签？

A: 性能优化器支持万级书签场景。1000 条书签的图谱构建 < 10 秒。大量书签会自动分批处理。

### Q: 如何切换语言？

A: PageWise 会自动检测浏览器语言设置。支持中文 (zh-CN) 和英文 (en-US)。

### Q: 学习进度数据会丢失吗？

A: 学习进度存储在 IndexedDB 中，持久化保存。使用浏览器清除数据功能可能会影响进度数据，建议定期导出备份。

### Q: 如何自定义智能集合规则？

A: 在智能集合面板中创建自定义集合，选择规则类型（标签/域名/文件夹/状态/时间范围/分类），组合多个规则（AND 逻辑）。规则可序列化保存。
