## 🔮 下一阶段 — LLM Wiki 知识编译系统

> 基于 Karpathy LLM Wiki 模式，将 PageWise 从「存储答案」升级为「编译知识」
> 参考: https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f

### Level 1 — 知识库 Markdown 导出（轻量级，改动最小）

- [x] **L1.1 知识库导出为 LLM Wiki 格式** — 将 IndexedDB 中的 Q&A 条目导出为结构化 Markdown 文件
  - 每条 Q&A → 独立 .md 文件（含 YAML frontmatter: title, tags, source_url, created）
  - 自动生成 `index.md`（按标签/分类分组）
  - 支持增量导出（只导出新增/修改的条目）
  - 导出路径: 用户选择的本地目录

- [x] **L1.2 实体/概念自动提取** — 导出时用 AI 自动识别 Q&A 中提到的实体和概念
  - 提取: 人名、工具名、框架名、API、概念术语
  - 生成独立的实体页（如 `entities/docker.md`）和概念页（如 `concepts/ci-cd.md`）
  - 实体页包含: 概述 + 相关 Q&A 列表 + 关联实体

- [x] **L1.3 交叉引用自动生成** — 导出的 Markdown 页面之间自动建立 `[[wikilinks]]`
  - 基于实体匹配: 如果两个页面提到同一实体，建立双向链接
  - 基于标签关联: 同标签的页面互相引用
  - 每个页面至少 2 个出站链接（孤立页面自动补链）

- [x] **L1.4 Git 集成** — 导出的 wiki 目录自动初始化为 Git 仓库
  - 每次导出自动 commit
  - 支持推送到用户指定的 GitHub 仓库
  - commit message 格式: `ingest: {N} new entries, {M} updated pages`

### Level 2 — 知识编译引擎（中等，需要迭代）

- [x] **L2.1 Q&A 自动分类** — 每次 AI 回答后，自动识别并标记涉及的实体/概念
  - 在现有 tags 基础上，增加 entity 和 concept 维度
  - IndexedDB schema 扩展: 新增 `entities` 和 `concepts` objectStore
  - 分类结果在知识面板中可视化显示

- [x] **L2.2 知识关联增强** — 不只是关键词匹配，基于实体/概念建立深度关联
  - 实体共现分析: 两个实体在同一 Q&A 中出现 → 建立关联
  - 概念层次关系: 识别 is-a / part-of / related-to 关系
  - 关联强度评分: 基于共现频率和语义相似度

- [x] **L2.3 矛盾检测** — 新回答与已有知识冲突时主动提示用户
  - 对比新 Q&A 与已有同主题 Q&A
  - 检测事实性矛盾（如版本号不同、API 变化）
  - 在侧边栏显示「⚠️ 知识冲突」提示，让用户确认

- [x] **L2.4 知识编译报告** — 每次 ingest 后生成编译报告
  - 新增/更新的页面数
  - 新发现的实体/概念
  - 新建立的交叉引用
  - 检测到的矛盾

- [x] **L2.5 增量编译** — 不是每次全量重新编译，只处理变化部分
  - 跟踪每个条目的编译状态
  - 新增条目只关联已有实体，不重新编译全部
  - 编译缓存: 已编译的页面不重复处理

### Level 3 — LLM Wiki 浏览器（完整，需要架构升级）

- [x] **L3.1 Wiki 浏览模式** — 在 PageWise 侧边栏新增「Wiki」标签页
  - 浏览 wiki 页面（实体/概念/对比/Q&A）
  - 支持 `[[wikilinks]]` 点击跳转
  - 页面内搜索
  - 按标签/类型筛选

- [ ] **L3.2 知识图谱可视化增强** — 在现有知识图谱基础上增加 wiki 视图
  - 节点类型区分: 实体(圆形) / 概念(方形) / Q&A(菱形)
  - 边类型区分: 引用(实线) / 关联(虚线) / 矛盾(红线)
  - 支持聚焦到单个节点查看关联子图
  - 导出图谱为图片

- [ ] **L3.3 自动 Ingest 流程** — 浏览任何网页时一键「编译进 wiki」
  - 点击「Ingest」按钮 → AI 分析页面内容 → 提取实体/概念 → 关联已有知识 → 生成 wiki 页面
  - 显示编译预览（将要创建/更新的页面列表）
  - 用户确认后执行编译
  - 编译结果在 Wiki 标签页中即时可见

- [ ] **L3.4 LLM Wiki 查询** — 对 wiki 提问而非对单个页面提问
  - 「Ask Wiki」模式: AI 读取整个 wiki 的知识来回答
  - 回答自动引用相关 wiki 页面
  - 有价值的回答可一键归档回 wiki

- [ ] **L3.5 Wiki Lint 工具** — 定期健康检查
  - 检测孤立页面、断裂链接、过时内容
  - 矛盾标记审查
  - 知识覆盖度报告（哪些主题缺少深度）
  - 一键修复建议

- [ ] **L3.6 服务器同步** — 与服务器上的 wiki 目录双向同步
  - 通过 WebDAV / Git / 自定义 API 同步
  - 支持离线编辑，联网后自动合并
  - Claude Code 在服务器上维护 wiki，PageWise 在浏览器中浏览
